## Root Cause

**Svelte child `onMount` fires before parent `onMount`.**

On `/admin/lrt`:
1. Child `onMount` → `initElectricSync()` → `getUkLrtCollection()` → Electric request with **no token** → 401
2. Parent (`admin/+layout.svelte`) `onMount` → `adminAuth.check()` → restores token from localStorage → **too late**

On `/browse`:
- No auth check anywhere — `adminAuth.check()` only exists in `admin/+layout.svelte`
- Token may exist in localStorage from prior `/auth/callback` visit, but nothing reads it

`adminAuth.check()` is synchronous (reads localStorage + decodes JWT), but placed inside `onMount` which runs **after** child components have already mounted and fired their own `onMount`.

## Architecture Context

- `legal.sertantai.com` nginx: `/api/*` → Phoenix backend, `/electric/*` → Electric direct (unused but exists)
- Frontend uses `/api/electric` which routes through `ElectricProxyController`
- Proxy extracts `Authorization` header → forwards to sertantai-auth Gatekeeper → Gatekeeper validates JWT + injects org-scoped WHERE → proxy forwards to Electric with `ELECTRIC_SECRET`
- All shapes require JWT — no public shape path exists, even for UK LRT reference data

### Phase 3: Gatekeeper succeeds but Electric returns 401 (FOUND)
- Gatekeeper returns 200 with `"status" => "success"` and validated shape
- Proxy forwards to Electric at `http://sertantai-legal-electric:3000`
- **Electric requires `ELECTRIC_SECRET` but backend container doesn't have it**
- `maybe_add_secret()` reads `Application.get_env(:sertantai_legal, :electric_secret)` → `nil`
- Electric container has `ELECTRIC_SECRET=0OyNVCK44mimWCu4Nf/21MCXnU2BUymm/z8Lp/VK/mo=`
- `.env` has namespaced `SERTANTAI_LEGAL_ELECTRIC_SECRET` but docker-compose didn't map it
- **Fix**: Added `ELECTRIC_SECRET=${SERTANTAI_LEGAL_ELECTRIC_SECRET}` to `infrastructure/docker/docker-compose.yml`
