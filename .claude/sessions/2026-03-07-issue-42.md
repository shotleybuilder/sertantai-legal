## Root Causes (Five Issues)

### 1. Auth timing (commit `8fc5b17`)
Svelte child `onMount` fires before parent `onMount`. Electric sync started before `adminAuth.check()` restored the JWT from localStorage.

### 2. Missing ELECTRIC_SECRET (infra commit `b19730e`)
`.env` had `SERTANTAI_LEGAL_ELECTRIC_SECRET` but `docker-compose.yml` didn't map it to `ELECTRIC_SECRET` on the backend container.

### 3. Columns dropped — the 400 errors (commits `97c6704`, `0671fcc`)
- Gatekeeper returns `columns` as a JSON array, but proxy only handled strings → columns silently dropped
- Gatekeeper doesn't return `columns` at all → added fallback to client-requested columns
- Without columns, Electric tries to sync generated columns → 400

### 4. Live-poll 401s (commit `27fad41`)
Some concurrent requests arrived at the backend without Authorization header despite all paths chaining through `electricFetchClient`. Fix: handle-based requests skip Gatekeeper entirely (handle proves prior authorization).

### 5. Progressive sync mode — no data despite 200s (commit `a8ad0fe`)
`progressive` maps to `on-demand` internally in TanStack DB. In on-demand mode:
- ShapeStream uses `log=changes_only` and `offset=now`
- Data is ONLY loaded via `loadSubset` → `fetchSnapshot`/`requestSnapshot`
- `loadSubset` is triggered by live queries, NOT by `collection.toArray`
- Pages use `collection.toArray` directly → loadSubset never called → no data
- Additionally, the proxy was stripping `subset__*` and `log` params needed by snapshot requests

Fix: Switch to `eager` mode (safe — WHERE clause limits to ~800 records, not 19K).

### 6. CORS missing expose_headers (commit `f652075`)
Corsica plug didn't expose `electric-*` headers to browser JS. Without `Access-Control-Expose-Headers`, the Electric client can't read response headers → `MissingHeadersError`.

### 7. Public tables requiring auth (commit `3297f6a`)
UK LRT, LAT, and annotations are public reference data but the proxy required JWT auth for all tables. Unauthenticated users got 401.

### 8. Browser HTTP caching (commit `8387138`)
Electric returns `cache-control: public, max-age=604800` (7 days). Browser caches proxied responses. Cached responses lack CORS headers → `MissingHeadersError` on subsequent requests. Also: Req decompresses response body but leaves stale `content-encoding`/`content-length` headers.

### 9. Double collection creation (browse page fix)
`lastWhereClause` used quoted column name (`"md_date"`) but `buildWhereFromFilters` produces unquoted (`md_date`). Mismatch triggered immediate collection recreation on every page load.

### 10. Unquoted string values in WHERE clauses (commit `f2ffa70`)
`buildWhereFromFilters` comparison operators (`greater_than`, `less_than`, `greater_or_equal`, `less_or_equal`) didn't quote string values. Date filters like `latest_amend_date >= 2024-01-01` were sent unquoted → Electric 400 Bad Request. The `is_before`/`is_after` operators already quoted correctly.
