## Root Causes (Three Issues)

### 1. Auth timing (commit `8fc5b17`)
Svelte child `onMount` fires before parent `onMount`. Electric sync started before `adminAuth.check()` restored the JWT from localStorage.

### 2. Missing ELECTRIC_SECRET (infra commit `b19730e`)
`.env` had `SERTANTAI_LEGAL_ELECTRIC_SECRET` but `docker-compose.yml` didn't map it to `ELECTRIC_SECRET` on the backend container. The proxy's `maybe_add_secret()` returned nil, Electric rejected the request.

### 3. Columns dropped — the actual 400 (commit `97c6704`)
The Gatekeeper returns `columns` as a JSON array (`["id","name",...]`). `forward_validated_shape` only handled `is_binary(cols)` — the list failed the guard, columns were **silently dropped**. Electric then tried to sync ALL columns including generated ones (`leg_gov_uk_url`, `number_int`) which it can't handle → 400.

Fix: added `is_list(cols)` clause that joins the list with `,` before forwarding to Electric.

### Phase 3: Shape recovery rewrite (commit `ef31108`)
- `return {}` from `onError` retries with stale internal offset/handle — Electric client doesn't reset

- Timestamp-based cooldown (30s) instead of boolean flag
- Deployed → still 400 even with offset=-1

### Phase 4: Columns silently dropped (commit `97c6704`)
- Tested Electric directly on server with `curl` — works with secret
- Tested Electric with no columns → **400** (generated columns can't be synced)
- Traced through proxy: Gatekeeper returns columns as list, `is_binary` guard fails, columns dropped
- Added `is_list(cols)` clause + test
- Deployed → awaiting verification
